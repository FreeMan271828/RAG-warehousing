- 创建日期：2026年2月
- 依赖模块：设备管理模块

---

## 一、设计概述

### 1.1 功能需求

智能小车（AGV）用于在仓库中运输出入库物料，主要功能包括：

1. **小车管理** - 小车基本信息、状态、电量监控
2. **任务调度** - 接收出入库工单任务，执行运输
3. **充电管理** - 电量监控、低电量自动充电、充电记录
4. **3D展示** - 实时在三维仓库模型中展示小车位置和状态

### 1.2 复用现有表

本模块复用现有数据库设计中的以下表：

- `device_category` - 设备类别（小车类别）
- `device_model` - 设备型号（小车型号）
- `equipment` - 设备表（小车设备，含3D坐标）
- `element` - 零件表（小车零件）

## 三、业务规则

### 3.1 电量管理

| 电量范围 | 状态 | 行为 |
|---------|------|------|
| 60% - 100% | 正常工作 | 可接受任务 |
| 20% - 60% | 电量不足 | 可接受任务，优先安排充电 |
| 10% - 20% | 低电量 | 停止接受新任务，返回充电站 |
| 0% - 10% | 即将耗尽 | 强制停止工作，自动返回充电站 |

### 3.2 充电规则

1. **自动充电** - 电量低于20%时自动返回充电站
2. **手动充电** - 管理员可手动安排充电
3. **充满停止** - 电量达到100%自动停止充电
4. **智能调度** - 优先使用空闲充电站

### 3.3 任务调度

1. 任务创建后进入任务池
2. 系统根据小车位置、电量、任务优先级自动分配
3. 同一时间一辆小车只能执行一个任务
4. 任务完成后小车自动返回空闲状态或充电

## 四、状态机设计

### 4.1 小车状态机

```
空闲(idle) <- -> 工作中(working)
     ↑            |
     |            ↓
  充电中  <- -> 返回中(returning)
   (charging)    (returning)
     ↓
   故障(fault)
```

### 4.2 任务状态机

```
待执行(pending) -> 执行中(executing) -> 已完成(completed)
                     ↓
                   已取消(cancelled)
                     ↓
                   失败(failed)
```

## 五、3D模型展示

### 5.1 小车3D模型

小车在3D仓库中的显示：

| 属性 | 说明 |
|-----|------|
| 位置 | 使用 equipment 表的 x_position, y_position, z_position |
| 显示 | 根据 agv_status 状态显示不同颜色 |
| 动画 | 移动时显示路径动画 |

### 5.2 状态颜色

| 状态 | 颜色 | 说明 |
|-----|------|------|
| 空闲(idle) | 绿色 | 可用 |
| 工作中(working) | 蓝色 | 执行任务中 |
| 充电中(charging) | 黄色 | 正在充电 |
| 返回中(returning) | 橙色 | 返回充电站 |
| 故障(fault) | 红色 | 故障停机 |

### 5.3 充电站3D模型

| 属性 | 说明 |
|-----|------|
| 位置 | 使用 charging_station 表的坐标 |
| 显示 | 固定位置，显示可用/使用中状态 |
